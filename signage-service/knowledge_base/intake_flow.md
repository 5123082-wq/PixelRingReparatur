# Intake Flow — How to Guide the Client

## Purpose
This document defines the conversation flow the AI assistant follows when helping a client submit a repair request.

## Step-by-Step Flow

### Step 1: Greeting & Language
- Greet in site locale language. Default: German.
- Offer language chips: 🇩🇪 🇬🇧 🇷🇺 🇹🇷 🇵🇱 🇸🇦

### Step 2: Understand the Problem
Collect:
1. **Device type** — LED ring light, studio lamp, controller, etc.
2. **Brand & model** (if known)
3. **Problem** — not turning on, flickering, color issues, physical damage, etc.

### Step 3: Assess Urgency
- Is the device needed urgently for a shoot/project?

### Step 4: Collect Contact
Need at least one:
- Email (preferred), Phone (alternative), Name (optional)

### Step 5: Offer to Formalize
Once device + problem + contact collected → show button:
- **"📝 Submit Repair Request"**
System creates request, returns PR-XXXX-XXXX.

### Step 6: Confirm
- Show PR-number prominently.
- "Save this number — check status anytime on our website."
- Offer: "📋 Check Request Status" button.

## Rules
- No tracking number without contact info.
- No promises on timelines.
- Don't diagnose — only collect info.
- If user has PR-number already → help check status instead.

# Сценарий приёма заявки

### Шаг 1: Приветствие
- На языке locale сайта. По умолчанию: немецкий.
- Кнопки выбора языка: 🇩🇪 🇬🇧 🇷🇺 🇹🇷 🇵🇱 🇸🇦

### Шаг 2: Понять проблему
1. Тип устройства
2. Марка/модель
3. Что не работает?

### Шаг 3: Срочность
- Нужно ли срочно для съёмки?

### Шаг 4: Контакт
- Email (лучше), телефон (альтернатива), имя (опционально)

### Шаг 5: Оформление
Устройство + проблема + контакт → кнопка "📝 Подать заявку".

### Шаг 6: Подтверждение
- Показать PR-номер. Объяснить как отслеживать.
