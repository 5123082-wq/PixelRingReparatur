# Design Principles

Last updated: 2026-04-13.

## Product Principles

- One accountable service company, not a marketplace.
- One entry point for requests with AI-assisted intake.
- Human specialists execute service after intake.
- Trust and clarity before visual expressiveness.
- German-first, multilingual-ready by default.

## UX Principles

- First screen must explain "what this service is" within seconds.
- Next action must be explicit and low-friction.
- User should always see at least one fast path to submit a request.
- Language switching must be visible and predictable.
- Customer-facing surfaces must not expose CRM internals.

## Visual Principles

- Professional service brand, not a contractor directory look.
- Strong hierarchy:
  - clear section titles
  - clear CTA zones
  - controlled density
- Use consistency in shape, spacing, and elevation to reduce cognitive load.
- Animation supports comprehension, not decoration.

## Content Principles

- Prefer direct service-company language over network/marketplace language.
- Explain AI as assistant, not replacement of company responsibility.
- Keep reassurance near critical conversion actions.
- Avoid ambiguous ownership wording that suggests open bidding or listings.

## Governance Principles

- Code is current-state source of truth.
- Docs must explicitly mark:
  - factual current behavior
  - intended target behavior
- Any unresolved mismatch must be recorded as a documented gap, not hidden.
